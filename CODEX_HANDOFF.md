## Task 1 Bug That Was Fixed
- The user reported this runtime error:
  - `Error using mars.run_simulations (line 22) Invalid parameter 'CaptureErrors' specified`
- Root cause:
  - `parsim` supports `CaptureErrors`
  - the sequential fallback path was calling `sim(sim_inputs(idx), 'CaptureErrors', 'on')`, which is invalid in that context
- Fix applied:
  - `automation/+mars/run_simulations.m` now uses:
    - `parsim(..., 'CaptureErrors', ...)` when parallel support exists
    - `sim(...)` inside `try/catch` when running sequentially
  - helper functions were updated so both result shapes are supported:
    - `automation/+mars/get_error_message.m`
    - `automation/+mars/get_sim_output_var.m`
    - `automation/+mars/get_simulation_end_time.m`
